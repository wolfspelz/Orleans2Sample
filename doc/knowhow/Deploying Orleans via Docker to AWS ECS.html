Deploying Orleans via Docker to AWS ECS

Alexander Prooks
Aug 28, 2018

Ms Orleans (https://dotnet.github.io/orleans/) was one of the latest addition to frameworks we use at https://apaleo.com/. We use Amazon ECS to host our services and we faced a problem: Orleans siloes should communicate with each other, but by default, ECS’s network does not span across docker images deployed to different hosts.

There is a special network mode which allows joining of a docker instance to elastic network, which spans across ECS and other EC2 instances. It works out of the box, but limitations on number of instances per cluster made it unusable for development and testing deployments

https://docs.aws.amazon.com/AmazonECS/latest/developerguide/task-networking.html

And here are the limits:

https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-eni.html#AvailableIpPerENI

TL;DR Use default ECS cluster’s networking, get host’s ip address and ports and send to Orleans to use for membership, but use docker’s internal ports as endpoints.

The actual workflow to make it all work would be following.

In order to get ip address call single endpoint: http://169.254.169.254/latest/meta-data/local-ipv4

Ports are a bit trickier. They can be loaded by enabling ECS and loading special metadata file. Check here how to enable it https://docs.aws.amazon.com/AmazonECS/latest/developerguide/container-metadata.html

Most probably you will have 2 ports exposed for silo and gateway. So port info will look like:

”PortMappings”: [ 
{ “ContainerPort”: 30000,
 “HostPort”: 32444, 
 “BindIp”: “”, 
 “Protocol”: “tcp” },
{ “ContainerPort”: 11111,
 “HostPort”: 32456, 
 “BindIp”: “”, 
 “Protocol”: “tcp” } ],

From here you can extract HostPort value by filtering container’s port .

Endpoint configuration will look like:

.Configure<EndpointOptions>(options =>
{
    // IP Address to advertise in the cluster
    // Must be host's port you have loaded in step 1
    options.AdvertisedIPAddress = IPAddress.Parse("10.0.1.1");
    // These 2 ports will be used by a cluster
    // for silo to silo communications
    options.SiloPort = 32456;
    // Port to use for the gateway (client to silo)
    options.GatewayPort = 32444;

    // Internal ports which you expose to docker
    options.GatewayListeningEndpoint = new IPEndPoint(IPAddress.Any, 30000);
    options.SiloListeningEndpoint = new IPEndPoint(IPAddress.Any, 11111);
})

Everything else can be configured like in this example https://github.com/dotnet/orleans/tree/master/Samples/2.0/docker-aspnet-core

I still have an open question of how to properly support health checks, and clustering when ECR decides to most of the cluster but that’s another story.
